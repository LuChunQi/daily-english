📖【完整对话】
─────────────────────────

**场景：** 项目经理 Sarah 在 Slack 上和开发 Tom 确认一个功能上线时间，同时通知设计师 Lisa 需要更新设计稿。

---

**A — Sarah (PM):** Hey Tom, quick heads up — the client wants the dashboard feature live by Friday. Are we on track for that?
> 嘿 Tom，提醒一下——客户希望 dashboard 功能周五上线。我们来得及吗？

**B — Tom (Developer):** Hmm, the backend is mostly done, but I'm still blocked on the API docs from the third-party vendor. Sent them a follow-up yesterday, no response yet.
> 嗯，后端基本完成了，但我还卡在第三方供应商的 API 文档上。昨天发了跟进邮件，还没回复。

**A — Sarah (PM):** Ugh, typical. Can you work around it for now? Maybe mock the API responses so we can at least test the frontend?
> 哎，老样子。你能不能先绕过去？比如 mock 一下 API 返回数据，至少前端能测起来？

**B — Tom (Developer):** Good call. I can put together a mock server by EOD today. That should unblock Lisa's design review too.
> 好主意。我今天下班前能搭好 mock server，这样 Lisa 的设计评审也能推进了。

**A — Sarah (PM):** Perfect. @Lisa — FYI, we're targeting Friday launch. Can you finalize the UI assets by Thursday COB?
> 完美。@Lisa 通知一下，我们目标周五上线。你能在周四下班前把 UI 资源定稿吗？

**C — Lisa (Designer):** Just saw this. Thursday COB is tight but doable. Tom, can you send me the updated component list so I know what changed?
> 刚看到。周四下班前有点紧但可以。Tom，你能发我更新后的组件清单吗？我得知道哪些改了。

**B — Tom (Developer):** Will do. Bumping this thread with the list shortly. Sarah, should I loop in David from QA for the Friday release?
> 好的，马上把清单发到这个 thread 里。Sarah，我要不要把 QA 的 David 也拉进来，准备周五的发布？

**A — Sarah (PM):** Yes please. Loop him in and set up a quick 15-min sync tomorrow morning. Let's not have any surprises on launch day. 👍
> 好的，把他拉进来，明天早上安排一个 15 分钟的快速同步。上线当天别出幺蛾子。👍

🎯【语块精讲】（8 个）
─────────────────────────

**1. heads up** — 提前告知、提醒
- 工作场景：Just a heads up, the meeting got moved to 3 PM.（提醒一下，会议改到3点了。）
- 日常场景：Heads up, there's no parking left out front.（提醒一下，前面没车位了。）

**2. blocked on** — 被……卡住、受阻碍
- 工作场景：I'm blocked on the data export — need DBA access.（我被数据导出卡住了——需要 DBA 权限。）
- 日常场景：I'm blocked on dinner plans — no one can agree on a restaurant.（晚饭没法定——谁也不同意去哪家。）

**3. work around (it)** — 绕过问题、找替代方案
- 工作场景：The API is down. Let's work around it with cached data for now.（API 挂了，先用缓存数据顶一下。）
- 日常场景：The highway's blocked. We'll work around it by taking side streets.（高速堵了，走小路绕过去。）

**4. Good call** — 好主意、说得好（赞同对方建议）
- 工作场景："Should we test on staging first?" "Good call."（"要不要先在 staging 测一下？""好主意。"）
- 日常场景："Let's order pizza instead of cooking." "Good call."（"咱们点披萨别做饭了。""好主意。"）

**5. EOD / COB** — EOD = End of Day（今天结束前）；COB = Close of Business（下班前）
- 工作场景：I'll have the report ready by EOD.（报告今天下班前给你。）
- 日常场景：（日常不太用，这是职场缩写）

**6. FYI** — For Your Information，供你参考（通知别人但不要求行动）
- 工作场景：FYI, the office will be closed next Monday.（通知一下，下周一办公室关闭。）
- 日常场景：FYI, that restaurant we talked about just got a Michelin star.（顺便说一下，我们说过的那家餐厅刚拿到米其林了。）

**7. Bumping this (thread)** — 顶帖、把消息提上来引起注意
- 工作场景：Bumping this — still waiting on approval for the budget.（顶一下——还在等预算审批。）
- 日常场景：Bumping this thread — who's bringing what to the potluck?（顶一下这个讨论——聚餐各带什么？）

**8. loop (someone) in** — 把某人拉进讨论/项目
- 工作场景：Let's loop in legal before we sign anything.（签字前把法务拉进来吧。）
- 日常场景：Should we loop in Jake for the road trip planning?（自驾游规划要不要把 Jake 拉进来？）

🔄【变造挑战】（3 题）
─────────────────────────

**1. 你在 Slack 上提醒同事明天有个重要的客户会议，问他准备好了没。**
💡 提示：用 heads up + on track

**2. 你的代码被另一个团队的接口卡住了，你在 Slack 上告诉项目经理这个情况，并说你有替代方案。**
💡 提示：用 blocked on + work around

**3. 你需要在群聊里把新来的实习生拉进一个项目的讨论，同时提醒大家明天下班前交周报。**
💡 提示：用 loop (someone) in + EOD / COB
