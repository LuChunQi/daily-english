# Day 02 变造挑战 — 参考答案

**1. 你是项目经理，周会上有人说"这个功能一周就能做完"，你觉得至少要三周。怎么礼貌地反驳？**

> I see your point, and I appreciate the optimism. But I think we need to drill down a bit — when you factor in testing, code review, and the API integration, I'd say three weeks is more realistic. Can we put together a detailed breakdown and circle back on Thursday?

---

**2. 你在会议上发现了一个潜在的技术风险，但不是你负责的领域。你想提醒团队，又不想越界。怎么说？**

> This might be outside my lane, but I wanted to flag a potential issue with the database migration. Have we thought about how it might affect the legacy reports? Just wanted to make sure we don't miss anything. Happy to take this offline with the data team if needed.

---

**3. 一个同事在会上提了个你觉得不太可行的方案，但对方很坚持。你怎么表达反对同时保持建设性？**

> I see where you're coming from, and I like the general direction. Playing devil's advocate for a second though — what happens if the vendor can't deliver on time? Do we have a backup plan? Maybe we could phase it: start with what we can control, and circle back on the vendor-dependent pieces once we have more clarity.
